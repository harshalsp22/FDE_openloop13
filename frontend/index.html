WCIS Algorithm
Weighted Component Intersection Scoring - custom online Bayesian-inspired incremental learning
01
First FDE — Prior Assignment
All components get base weight +2.0. All are suspects. Analogous to a uniform prior in Bayesian inference.
02
Repeat FDE — Intersection Update
WCIS computes NewSet ∩ PastFDE[i] for every past event. Common components get weight 3.0 + (overlaps−1)×1.5.
03
Risk Scoring — Additive Model
For any new drug: Risk = Σ score(c) per component. LOW < 3, MODERATE 3–7, HIGH ≥ 7.
04
Safe Alternative Search
Scans 1,600-medicine database. Matches same Used_For category. Returns drugs with zero risky components.
1st FDE → +2.0 per component
1 overlap → +3.0
2 overlaps → +4.5
3 overlaps → +6.0
no overlap → +0.5
LOW <3 · MOD 3–7 · HIGH ≥7
Why Not Random Forest / SVM / XGBoost?
Standard classifiers need hundreds of labelled training samples per patient before they can predict. FDE is rare — a patient may have only 1–3 lifetime events.

WCIS is an online incremental algorithm — it learns meaningfully from the very first event and is fully per-patient. Complexity: O(E × C) per update.

WCIS Algorithm
Weighted Component Intersection Scoring — custom online Bayesian-inspired incremental learning
01
First FDE — Prior Assignment
All components of the reacting drug get base weight +2.0. No narrowing yet — all are suspects. Analogous to a uniform prior in Bayesian inference.
02
Repeat FDE — Intersection + Likelihood Update
WCIS computes NewSet ∩ PastFDE[i] for every past FDE event. Common components get weight 3.0 + (overlaps−1)×1.5. More overlaps = stronger culprit evidence.
03
Risk Scoring — Additive Model
For any new drug: Risk = Σ score(c) for each component c, lightly normalized for large component counts. Thresholds: LOW < 3, MODERATE 3–7, HIGH ≥ 7.
04
Safe Alternative Search
Scans 1,600-medicine database. Matches same Used_For category. Returns drugs with zero risky components (score ≥ 3.0) — safe swaps with identical therapeutic effect.
1st FDE → +2.0 per component
1 overlap → +3.0
2 overlaps → +4.5
3 overlaps → +6.0
no overlap → +0.5
LOW <3 · MOD 3–7 · HIGH ≥7
🧮 Why Not Random Forest / SVM / XGBoost?
Standard classifiers need hundreds of labelled training samples per patient before they can predict. FDE is rare — a patient may have only 1–3 lifetime events. RandomForest would have zero data to build trees from.

WCIS is an online incremental algorithm — it learns meaningfully from the very first event and improves with each subsequent one. It is fully per-patient — Patient A's penicillin score never contaminates Patient B's model. A population-trained RandomForest cannot achieve this level of personalization.

Complexity: O(E × C) per update where E = past FDE events, C = components per drug. Efficient and deterministic.
